BigData Architecture

HETEROGENEOUS GEOSPATIAL DATA SOURCES

Our system is able to work, join, and process togegher different datasources.
On one hand, it can access geospatial data from official websites such as IGN (National Geographic Institute of Spain), GADM (Database of Global Administrative Areas), or ArcGIS from Esri OpenData portal, for example. Maps are more or less statical if we consider they're not usually updated.
On the other hand, some geospatial is refreshed every day, or some times in a day, losing previous information by updating. That's what happens with Traffic Incidence.
Our system not only process its information and cross with other Non-spatial data such as INE (National Statistics Institute of Spain), but also is able to store for further DataMining prediction studies like Temporal Series.

  • Static geo data
  • Non geo data
  • Dynamic geo data

IGN, GADM

These portals provides data from National Topographic Base (BTN) and National Cartographic Base (BCN), in projections WGS84, ETRS89 and REGCAN95.
They've been downloaded according to their disclaimers and they conform the geospatial dataframe base which the rest of data are added, in order to do geo-referenced studies.

INE

This portal is one of the most important data sources in Spain. It contains a lot of useful information but it's not geo-referenced. So, the system has to manage it in order to add it to geospatial dataframe. Concretely, we'll work with towns population, household expenditures in public and private transport.

Esri OpenData

From this portal we'll use statical data such as geo-referenced inter-urban bus stops, and furthermore dynamic data such as traffic incidents, which are daily updated. They are also used as base to add data from INE tables/datasheets/CSV.
